JavaScript is a lightweight, cross-platform, and single-threaded object-oriented programming language. It is used for interactive web applications. It supports both client-side and server-side development. It is an interpreted language, which means it executes code line by line.


History of JavaScript


  1. 1995 - Brendan Eich creates JavaScript in 10 days at Netscape.
  2. 1996 - Microsoft releases JScript for Internet Explorer.
  3. 1997 - ECMAScript (ES1) is standardized.
  4. 1999 - ES3 introduces Regex & exception handling.
  5. 2005 - AJAX revolutionizes web interactivity.
  6. 2006 - jQuery simplifies JavaScript.
  7. 2009 - ES5 brings JSON support, map(), filter(), reduce().
  8. 2010 - AngularJS popularizes front-end frameworks.
  9. 2013 - React.js changes UI development Widely.
  10. 2014 - ECMAScript updates become annual.
  11. 2015 - ES6 (ECMAScript 2015) introduces let, const, classes, modules, and arrow functions.
  12. 2016 - Vue.js gains popularity as an alternative to React.
  13. 2017 - ES8 adds async/await, making asynchronous coding easier.
  14. 2019 - ES10 introduces optional catch binding and flatMap().
  15. 2020 - Deno (by Ryan Dahl) challenges Node.js with better security.
  16. 2021 - ES12 adds logical assignment operators (&&=, ||=, ??=).
  17. 2022 - ES13 introduces at() for arrays and top-level await.
  18. 2023 - ES14 brings Array grouping (groupBy()).
  19. 2024 - React Server Components (RSC) revolutionize web app performance.
  20. 2025 - JavaScript continues evolving, integrating more AI-driven features & WebAssembly advancements.


Ways to Add JavaScript to a Web Page


When working with files for the web, JavaScript needs to be loaded and work together with HTML. You can either write JavaScript directly inside the HTML file or put it in a separate file that the browser will load with the HTML. You can add JavaScript code in an HTML document by using the dedicated HTML tag <script>, which wraps around the JavaScript code. Here, I will guide you through the process of adding JavaScript to your web pages.

The <script> tag can be placed in the <head> section or in the <body> section of your HTML, depending on when you want the JavaScript to load.


The <script> tag inside the document <head>


  1. The browser starts parsing HTML and building the DOM.
  2. When it finds a script, it stops parsing and starts downloading the script.
  3. Then executes the script before continuing.


This approach blocks HTML parsing, causing a delay in rendering. Avoid unless necessary.


Example:

<!DOCTYPE html>
<html>
<head>
<script>
console.log('Hello from internal JS');
</script>
</head>
<body>
<!-- content -->
</body>
</html>


The browser will start reading the page from the top. It will get the <script> tag inside the <head>, it stops reading the page and run the JavaScript inside this <script>. You will see this message in the browser‚Äôs console: "Hello from internal JS". Then the browser will continue reading the page.


The <script> tag at the end of the <body>

It helps to run the JavaScript code after the HTML content is loaded.


  1. The browser downloads and builds the DOM.
  2. When it finds a script, it starts downloading the script.
  3. Then executes the script right after.


Example:

<body>
<h1>Hello World</h1>
<button onclick="sayHello()">Click Me</button>

<script>
function sayHello() {
alert('Hello!');
}
</script>
</body>


The browser will start reading the page from the top and continue until the entire page is read. After building the DOM, it will reach the <script> tag at the end of the body. Then, it will download and execute the JavaScript code. When we click the button, it will show an alert saying, "Hello!"


The <script async> tag at the end of the <body>


  1. The browser keeps reading the HTML without waiting
  2. It starts downloading the script in parallel without pausing the DOM.
  3. The script runs as soon as it‚Äôs downloaded.


Best for scripts that don‚Äôt depend on the page content, like analytics or ads. Avoid using for scripts that need to interact with the page elements.


Example:

<head>
<script src="analytics.js" async></script>
</head>
<body>
<h1>Hello World</h1>
</body>



The <script defer> tag at the end of the <body>


  1. The browser reads the HTML and downloads the script in the background.
  2. The script only runs after the entire page is fully loaded and the DOM is ready


Example:

<head>
<script src="app.js" defer></script>
</head>
<body>
<h1>Hello World</h1>
<button id="btn">Click Me</button>
</body>


Great for scripts that need to work with the page content but shouldn‚Äôt delay the loading of the page.
